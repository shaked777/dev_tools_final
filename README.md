## 📝 ב. מה נדרשנו לעשות
במסגרת מטלת הסיום, נדרשנו להקים אתר Drupal ייעודי עבור מילון מושגי הקורס לסמסטר ב' התשפ"ו. 
הדרישות המוגדרות במסמך ההנחיות המקוריות "2026B.devtools.final_project_DRUPAL.setup.pdf" הן:
* **מערכת Drupal:** הינה מערכת מתקדמת לניהול תוכן (CMS) המאפשרת בניית עמודי אתרים, ונחשבת לאחד השירותים הנפוצים בעולם לאתרי בלוג ותוכן לצד וורדפרס וג'ומלה. היישום עצמו מבוסס קוד פתוח וכתוב בשפת PHP.
* **בסיס הנתונים:** על מנת לשמור את התכנים והגדרות האתר בצורה קבועה, נדרשנו לבחור ולשמור את הנתונים בבסיס נתונים מסוג MySQL או PostgreSQL. לטובת פרויקט זה נבחר שירות PostgreSQL.
* **סביבת העבודה:** את שני חלקי הפרויקט (האתר ובסיס הנתונים) נדרשנו להפעיל ולנהל בצורה מבודדת ומנוהלת באמצעות שירות ה-Docker.

## ⚙️ ג. מה עשינו (פירוט שלבי העבודה)
במהלך הפרויקט ביצענו את השלבים המעשיים הבאים:
1. **הכנת הרשת והתשתיות:** הגדרנו ופתחנו רשת Docker פנימית ייעודית בשם `drupal-net` על מנת שהקונטיינרים השונים יכירו ויקיימו תקשורת מאובטחת ביניהם.
2. **הקמת בסיס הנתונים (PostgreSQL):** הרמנו קונטיינר המבוסס על הגרסה העדכנית ביותר של ה-Image הזמין להורדה מ-Docker Hub. הגדרנו את פורט ברירת המחדל 5432 מול מכונת הלינוקס שעליה הוא רץ וקבענו את סיסמת משתמש-העל (root) לערך המדויק `my-secret-pw` כפי שנדרש. השתמשנו במשתני מערכת (Environment Variables) להגדרת פרטי הגישה הפנימיים לשם בסיס הנתונים והמשתמשים.
3. **הקמת יישום Drupal:** הרצנו קונטיינר רשמי של דרופל המחובר לאותה הרשת הפנימית שהגדרנו, תוך חשיפת פורט 8080 במכונה המקומית המפנה לפורט 80 הפנימי בתוך הקונטיינר.
4. **קונפיגורציה והזנת תוכן:** התחברנו למערכת דרך הדפדפן בכתובת `http://localhost:8080`. הגדרנו את האתר לעבודה בשפה המבוקשת וקישרנו אותו לבסיס הנתונים באמצעות שם קונטיינר ה-DB, הפורט ופרטי הגישה. קבענו את שם האתר בפורמט המבוקש המכיל את שמות כל חברי הצוות ("האתר של שקד גבאי, לידור אבן ואייל גולדשטיין"), הקמנו חשבון מנהל עם שם המשתמש `demoadmin` והסיסמה `secretpass`, ופתחנו חשבונות משתמש נפרדים לכל שותף בצוות. לאחר מכן, ביצענו Login והוספנו תכנים לאתר (3 דפי Basic Pages) מתוך מילון המושגים של הקורס במודל.
5. **פתרון בעיית ה-Settings בשחזור:** במהלך בדיקת השחזור במכונת לינוקס אחרת, זיהינו כי שחזור הדאטהבייס לבדו גורם לאתר לחזור למסך ההתקנה הראשוני מפני שהוא לא מכיר את הגדרות החיבור. כדי לפתור זאת, גיבינו וסנכרנו באופן מובנה גם את קובץ ה-`settings.php` של דרופל, והגדרנו מנגנון המעניק לו הרשאות בעלות מתאימות עבור שרת ה-Web הפנימי (`chown www-data`) כדי להבטיח עליית אתר מהירה ומלאה ללא התקנה מחדש.

## 🛠️ ד. באיזה טכנולוגיות השתמשנו
* **Docker & Docker Network:** ליצירת סביבת רכיבים מבודדת ורשת וירטואלית משותפת.
* **Drupal CMS:** אפליקציית ניהול תוכן מתקדמת מבוססת PHP שעליה נבנה האתר ומילון המושגים.
* **PostgreSQL:** בסיס הנתונים שנבחר לאחסון וניהול כלל הטבלאות והתכנים של המערכת.
* **Bash Scripting:** לכתיבת סקריפטי האוטומציה לניהול מלא של מחזור החיים של הפרויקט.
* **Git & GitHub:** לניהול גרסאות, שיתוף קבצים וסנכרון העבודה בין חברי הצוות.

## 📖 ה. מדריך Step-By-Step למשתמש (איך לשחזר ולנקות את האתר)

מדריך זה מיועד להרצה על גבי מכונת לינוקס (Ubuntu וכדומה) נקייה לחלוטין.

### שלב 1: שכפול ה-Repository מה-Git (git clone)
יש לפתוח את הטרמינל במכונה החדשה ולשכפל את המאגר המשותף:
```bash
git clone https://github.com/shaked777/dev_tools_final.git
cd dev_tools_final
```

### שלב 2: מתן הרשאות הרצה לסקריפטים
לפני הפעלת האוטומציה, יש להעניק לקבצי ה-Bash הרשאות ריצה מתאימות במערכת הלינוקס:
```bash
chmod +x scripts/*.sh
```
*(שים לב: במידה ושם התיקייה אצלך במאגר מוגדר באות גדולה, יש להקפיד על נתיב מדויק: `Scripts/*.sh`)*.

### שלב 3: הקמת התשתית האוטומטית (setup.sh)
נפעיל את סקריפט ההקמה, שתפקידו ליצור את רשת ה-Docker הפנימית המשותפת ולהוריד ולהריץ את הקונטיינרים של דרופל ובסיס הנתונים באופן אוטומטי:
```bash
./scripts/setup.sh
```
*ניתן לוודא ששני השירותים רצים ברקע על ידי הרצת הפקודה `docker ps`.*

### שלב 4: שחזור הנתונים וההגדרות (restore.sh)
כדי להזריק את כלל המידע המגובות מקובץ ה-SQL לתוך בסיס הנתונים החדש, ולסנכרן את קובץ הקונפיגורציה המבני שמונע את לולאת ההתקנה מחדש של דרופל, הרץ את סקריפט השחזור:
```bash
./scripts/restore.sh
```
הסקריפט יטען את הנתונים ויבצע אוטומטית הפעלה מחדש (restart) לשירות הדרופל כדי להחיל את השינויים במערכת.

### שלב 5: פתיחת האתר והגדרתו בדפדפן
פתח את הדפדפן במכונה וגש לכתובת הבאה:
👉 **`http://localhost:8080`**

האתר ייפתח באופן מיידי על דף הבית המשוחזר, תחת הכותרת *"האתר של שקד גבאי, לידור אבן ואייל גולדשטיין"*, ויציג את כל דפי המושגים שהוזנו מראש. ניתן להתחבר למסך הניהול באמצעות לחצן הכניסה עם פרטי ה-Admin הרשמיים: משתמש `demoadmin` וסיסמה `secretpass`.

### שלב 6: ניקוי סביבת העבודה ופינוי המידע
בסיום הבדיקה, על מנת לנקות את כל הקבצים שהוגדרו, למחוק לחלוטין את הקונטיינרים של דרופל ובסיס הנתונים, להסיר את ה-Volumes ואת תמונות ההרצה (Images) מהמחשב, הרץ את סקריפט הניקוי:
```bash
./scripts/cleanup.sh
```

## 💡 ו. מידע נוסף לסיוע למשתמשים
* **גיבוי ידני של בסיס הנתונים:** במידה ותצטרך לייצר קובץ גיבוי SQL עדכני מחוץ לסקריפט האוטומטי, תוכל להשתמש בכלי ה-`pg_dump` הרשמי המופעל ישירות על קונטיינר הדאטהבייס:
  ```bash
  docker exec -i postgres-db pg_dump -U drupal_user drupal_db > drupal_db_backup.sql
  ```
* **פתרון התנגשות פורטים (Port 5432):** אם סקריפט ההקמה נכשל ומציג את השגיאה `bind: address already in use`, המשמעות היא שקיים שירות PostgreSQL מקומי שרץ ישירות על המחשב המארח ומחזיק את הפורט. יש לעצור אותו באמצעות פקודת המערכת `sudo systemctl stop postgresql` (בלינוקס) או `brew services stop postgresql` (ב-Mac) לפני הרצת ה-Setup מחדש.
